# ブランドストーリー要約版の設計検討

## 2つのアプローチ

### アプローチ1: ストーリーの冒頭だけを入れる（1ブロック）

#### 構成
メインページに体験談の冒頭部分（最初の1-2段落）を配置し、「続きを読む」で詳細ページへ。

#### 具体例

**メインページ（要約版）:**
```
告知の重さは、生存率の数字では測れません

10年前、私は「がんです」と告げられました。
たとえ予後が良いとされる種類であっても、ステージが低くても、関係ありませんでした。
「声が出なくなるかもしれない」という漠然とした、けれど鋭い不安。

[続きを読む] → /story
```

#### メリット

1. **自然な流れ**
   - ストーリーの導入として自然
   - 読者の興味を引く（フックとして機能）
   - 小説や記事のような読み進める体験

2. **実装が簡単**
   - 既存のテキストをそのまま使用
   - 追加の要約文を書く必要がない
   - メンテナンスが容易

3. **感情的なつながり**
   - 冒頭のインパクトが強い
   - 共感を呼びやすい
   - 「続きが気になる」という心理を利用

4. **SEO的にも有効**
   - 冒頭部分がメインページに残るため、検索エンジンがインデックス可能
   - キーワードが含まれる

#### デメリット

1. **情報が不完全**
   - メインページだけでは全体像が分からない
   - コンセプト（「整理」ではなく「預ける」）が伝わらない
   - ストーリーの結論が見えない

2. **「続きを読む」への依存**
   - 遷移しないと全体が理解できない
   - メインページだけでは価値提案が不明確

3. **ブランド哲学との整合性**
   - 「静かな書斎」というブランド哲学からすると、情報を断片的に提示するのは適切か？

---

### アプローチ2: 本当の要約を入れる（独立した要約ブロック）

#### 構成
メインページに体験談とコンセプトの両方を要約した独立したブロックを配置。

#### 具体例

**メインページ（要約版）:**
```
不安をここに預けて、あなたが思う大事なことに集中する。

10年前、私は「がんです」と告げられました。その情報の波に飲まれて、自分の人生の歩幅を取り戻すまでに、2年という長い月日を費やしてしまいました。

多くの人は「情報を整理して、落ち着きましょう」と言います。でも、パニックの最中にいる人にとって、整理はさらなる「重荷」でしかありません。

「整理」ではなく、荷物を「預ける」という選択。ライフデザイン・ロジスティクス（ライフロジ）。それは、あなたが抱えている重い荷物を一度すべて降ろし、専門的な仕分けを仕組み（ロジ）に任せるという考え方です。

[ブランドストーリーを読む] → /story
```

#### メリット

1. **完結性**
   - メインページだけでも全体像が理解できる
   - 価値提案が明確に伝わる
   - 遷移しなくても核心が分かる

2. **ブランド哲学との整合性**
   - 「静かな書斎」のように、情報が整理されて提示される
   - 事実を淡々と記すスタイル
   - ユーザーが必要な情報を選択できる

3. **SEOの観点**
   - 要約版にキーワードが含まれる
   - メインページだけで価値提案が伝わる

4. **ユーザーの選択**
   - 要約で十分な人は詳細ページに遷移しない
   - 深く知りたい人は詳細ページへ
   - ユーザーのニーズに応じた情報提供

#### デメリット

1. **実装の複雑さ**
   - 要約文を別途作成する必要がある
   - 要約版と詳細版の一貫性を保つ必要がある
   - メンテナンスが2箇所必要

2. **感情的なインパクト**
   - 要約では体験談の細かい描写が失われる
   - 冒頭のインパクトが弱まる可能性

3. **重複のリスク**
   - 要約版と詳細版で内容が重複
   - 詳細ページを読む際に「既に読んだ」感が出る可能性

## 推奨案の比較

### azukのブランド哲学を考慮

**ブランド哲学**: 「Neutral Kindness（ドライだけど、暖かい）」
- 過剰に励まさない
- 事実を淡々と記す
- 静かな「書斎」や「金庫」のように、常に整然とそこに在り

### 推奨: **アプローチ2（本当の要約）**

**理由:**

1. **ブランド哲学との整合性**
   - 「静かな書斎」のように、情報が整理されて提示される
   - 事実を淡々と記すスタイルに合致
   - ユーザーが必要な情報を選択できる構造

2. **完結性**
   - メインページだけでも価値提案が理解できる
   - 遷移しなくても核心が伝わる
   - ユーザーを尊重する姿勢

3. **実用性**
   - メインページは機能説明とコンバージョンに集中
   - ストーリーは「深く知りたい人向け」として明確に分離
   - 各ページの役割が明確

4. **SEOの観点**
   - 要約版にキーワードが含まれる
   - メインページだけで価値提案が伝わる

### 代替案: **アプローチ1（冒頭のみ）**

もし感情的なインパクトを重視したい場合は、アプローチ1も検討可能です。

## デザインの考慮事項

### レイアウト

1. **セクションの配置**
   - HeroSectionの直後、またはFeatureSectionの前
   - 背景色は`bg-background`（生成り）で統一

2. **リンクのデザイン**
   - 控えめなデザイン（ブランド哲学に沿って）
   - 過度に目立たせない
   - 「続きを読む」または「ブランドストーリーを読む」

3. **タイポグラフィ**
   - 見出し: 24px, Bold
   - 本文: 16px, Regular, 行間1.6-1.8
   - リンク: 16px, Semibold

### レスポンシブ

- モバイルでも読みやすい余白
- リンクはタップしやすいサイズ
