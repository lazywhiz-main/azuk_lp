# リリースノート Markdownファイル仕様

## 📋 概要

このディレクトリに配置される`.md`ファイルは、azukアプリのリリースノートを記述するためのMarkdownファイルです。

## 📝 ファイル命名規則

- ファイル名: `v{X.Y.Z}.md`（例: `v1.0.1.md`）
- バージョン番号はセマンティックバージョニングに従う

## 📄 ファイル構造

### 必須セクション

1. **タイトル（H1）**
   ```markdown
   # azuk v1.0.1 リリースノート
   ```

2. **リリース日**
   ```markdown
   **リリース日**: 2026年1月XX日
   ```

3. **短い説明（App Store用）**
   ```markdown
   ## 短い説明（App Store用）
   
   記録、ロードマップ、家族共有機能を追加しました。
   ```

4. **詳細説明（App Store用）**
   ```markdown
   ## 詳細説明（App Store用）
   
   azukは、がん治療中の記録と情報管理をサポートするアプリです。
   
   ### 主な機能
   
   - 治療記録の作成・管理
   - 治療の流れ（目安）の表示
   - AIによる記録分析
   - 家族との情報共有
   - 信頼できる医療情報へのアクセス
   ```

5. **短い説明（Google Play用）**
   ```markdown
   ## 短い説明（Google Play用）
   
   記録、ロードマップ、家族共有機能を追加しました。
   ```

6. **詳細説明（Google Play用）**
   ```markdown
   ## 詳細説明（Google Play用）
   
   azukは、がん治療中の記録と情報管理をサポートするアプリです。...
   ```

## ✍️ Markdown記法ルール

### 見出し

- **H1**: `#` - タイトル用
- **H2**: `##` - セクション見出し
- **H3**: `###` - サブセクション見出し

**重要**: `【】`で囲まれたテキストは見出しとして認識されません。必ず`###`を使用してください。

```markdown
❌ 誤: 【主な機能】
✅ 正: ### 主な機能
```

### リスト

- **順序なしリスト**: `- `（ハイフン + スペース）を使用
- **順序ありリスト**: `1. `（数字 + ピリオド + スペース）を使用

**重要**: `・`（中点）で始まる行はリストとして認識されません。必ず`- `を使用してください。

```markdown
❌ 誤:
・治療記録の作成・管理
・治療の流れ（目安）の表示

✅ 正:
- 治療記録の作成・管理
- 治療の流れ（目安）の表示
```

### 段落と改行

- **段落**: 空行で区切る
- **段落内の改行**: 単一の改行は無視されます。改行が必要な場合は、空行で段落を分けるか、`<br />`を使用

```markdown
段落1の内容。

段落2の内容。
```

### 強調

- **太字**: `**テキスト**` または `__テキスト__`
- **斜体**: `*テキスト*` または `_テキスト_`

### 区切り線

```markdown
---
```

## ⚠️ 注意事項

1. **標準のMarkdown記法を使用**: `【】`や`・`などの独自記法は使用しない
2. **一貫性**: すべてのリリースノートで同じ構造と記法を使用
3. **改行**: 段落は空行で区切る。単一の改行は無視される
