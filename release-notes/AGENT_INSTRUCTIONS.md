## 📋 実装概要

azukアプリ内からWebViewで表示されるリリースノートページを実装してください。

### URL

- **本番環境**: `https://www.azuk.jp/release-notes`
- **開発環境**: 適宜設定

### データソース

以下のディレクトリに`.md`ファイルが配置されています：
```
azuk_lp/release-notes/
├── README.md        # 実装ガイド（詳細な技術仕様）
├── v1.0.1.md        # リリースノート（v1.0.1）
└── ...
```

---

## ✅ 実装要件

### 機能要件

1. **リリースノート一覧ページ** (`/release-notes` または `/release-notes/index.html`)
   - バージョン一覧を表示（新しい順）
   - 各バージョンのサマリーを表示
   - 各バージョンの詳細ページへのリンク

2. **リリースノート詳細ページ** (`/release-notes/v{X.Y.Z}.html`)
   - 各バージョンのリリースノート全文を表示
   - Markdown形式のコンテンツをHTMLに変換して表示

3. **レスポンシブ対応**
   - モバイル（アプリ内WebView）対応を優先
   - デスクトップ対応

### デザイン要件

**azukのデザインシステムに準拠**（詳細は `release-notes/README.md` を参照）

#### カラーパレット

```css
--color-primary: #556AAB;
--color-text-primary: #1B222E;
--color-text-secondary: #4A5568;
--color-text-tertiary: #718096;
--color-bg-base: #FFFFFF;
--color-bg-secondary: #FAF9F6;
--color-border: #E2E8F0;
```

#### トーン

**「ドライだけど暖かい（Neutral Kindness）」**
- 事実のみを提示
- 余計な説明や誘導を避ける
- 適切な余白で読みやすさを確保
- シンプルな構造

---

## 🔧 実装方法

### 推奨: 静的HTML生成

1. **MarkdownをHTMLに変換**
   - `marked` または `markdown-it` を使用
   - ビルド時に変換

2. **HTMLファイルを生成**
   - `index.html` - リリースノート一覧
   - `v{X.Y.Z}.html` - 各バージョンの詳細ページ

3. **デプロイ**

### 実装例

詳細な実装例とサンプルコードは `release-notes/README.md` を参照してください。
